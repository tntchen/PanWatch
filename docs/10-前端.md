# 10. 前端（frontend/）

## 模块职责一句话总结

PanWatch 前端是一个 React 18 + TypeScript + Vite 5 + Tailwind 3 的单页应用（SPA，含 PWA 支持），通过统一封装的 `@panwatch/api` 包与后端 FastAPI（`/api` 前缀）交互，提供首页大盘、持仓管理、机会/信号、模拟盘、价格提醒、Agent 管理、历史、数据源、设置等九大页面，以及 TradingAgents 深度分析阅读页。

## 工程结构与构建

- `frontend/package.json`：依赖较少且克制——React 18、react-router-dom v6、Radix UI 系列（dialog/select/switch/tabs/toast 等）、tailwindcss + tailwind-merge + class-variance-authority、lucide-react 图标、react-markdown + remark-gfm（渲染 AI 输出）、html-to-image（分享卡导 PNG）。无状态管理库、无数据请求库（React Query/SWR 均无）。
- `frontend/vite.config.ts`：dev 端口固定 `5183`（strictPort），`/api` 代理到 `http://127.0.0.1:8000`；定义 4 个路径别名：`@` → `src`，`@panwatch/api|base-ui|biz-ui` → `packages/*/src`（**别名直接指源码，不走构建产物**，所以改 packages 代码即时生效）。
- `frontend/tailwind.config.js`：darkMode 用 class 策略；颜色全部映射到 `hsl(var(--xxx))` CSS 变量（在 `src/index.css` 的 `:root`/`.dark` 定义）；`content` 覆盖 `src/` 和 `packages/`；内置 A 股配色 `stock.up=#E53935`（红涨）/`stock.down=#43A047`（绿跌）。
- `frontend/index.html` + `public/sw.js` + `manifest.json`：PWA（"盯盘侠"），Service Worker 只做静态资源缓存；**lightweight-charts 通过 CDN（unpkg，失败回退 jsdelivr）以全局脚本方式加载**，K 线图组件从 `window.LightweightCharts` 取值——离线/内网部署会拿不到图表库。
- 构建：`pnpm build` = `tsc -b && vite build`，产物为 `frontend/dist`（Docker 构建时拷贝到 `./static` 由后端托管）。

## packages 子包（pnpm workspace，三层架构）

`packages/README.md` 明确要求：页面统一从 `@panwatch/api` 发请求，**禁止在页面中直接 `fetch`**。

### `@panwatch/api`（领域 API 层，约 1800 行）

- `client.ts` 核心：所有请求走 `fetchAPI(path)`，自动拼 `/api` 前缀、注入 `Authorization: Bearer <token>`、统一解包后端 `{code, data, message}` 信封（`code !== 0` 抛错）、默认 20s 超时（`timeoutMs` 可调）、**401 自动 `logout()` 跳登录页**。token 存 `localStorage` 的 `token`/`token_expires` 两个键。
- 按领域分文件，每个导出一个 `xxxApi` 对象 + 对应 TypeScript interface：
  - `auth.ts`：登录/初始化设置（`/auth/login|setup|status`）。
  - `stocks.ts`：自选股 CRUD、个股绑定 Agent 配置、手动触发 Agent（触发超时放宽到 120s）。
  - `dashboard.ts`：首页数据（指数、市场状态、组合汇总、批量行情、`/dashboard/overview` 聚合接口、AI 精选 `/dashboard/curate`、盘前/收盘简报 `/dashboard/brief`、盘中扫描 `/agents/intraday/scan`）。
  - `recommendations.ts`（568 行，最大）：入场候选（entry-candidates）、策略信号（strategy-signals）、策略目录/因子/政权（regimes）/风险快照/权重历史、反馈统计——对应"机会"页全部数据面。
  - `portfolio.ts`：组合诊断（HHI 集中度）、基准对比、收益归因、AI 体检。
  - `paper-trading.ts`：模拟盘账户/持仓/交易/指标/通知设置/手动扫描。
  - `tradingagents.ts`：深度分析触发（复用 stocks trigger 接口）、进度轮询（`/agents/runs/:trace_id/progress`）、预算查询、运行中任务查找、**PDF 下载是唯一的原生 `fetch` 例外**（需要拿 blob 和 Content-Disposition 文件名）。
  - `insight.ts`：个股行情/K线/建议/新闻/加仓评估/公告评估。
  - 其余：`chat.ts`（AI 对话）、`discovery.ts`（热股/板块）、`factors.ts`（因子权重）、`health.ts`（系统自检）、`home.ts`（提醒命中/待办）、`datasources.ts`（数据源重置）、`app.ts`（版本）。
  - `types.ts`：AIService/AIModel/NotifyChannel/DataSource 等共享类型。

### `@panwatch/base-ui`（基础 UI，shadcn 风格）

- `cn.ts`：`clsx + tailwind-merge` 的 `cn()` 工具（`src/lib/utils.ts` 从这里 re-export）。
- `components/ui/`：button、input、label、switch、badge、dialog、select、tabs、popover、hover-popover、skeleton、calendar、toast（自实现的轻量 Context 版 toast，`useToast()` 全局可用，`ToastProvider` 在 `main.tsx` 挂载）。

### `@panwatch/biz-ui`（业务组件）

- `analysis-sections.ts`：把深度分析 `raw_data` 组装成统一 section 列表（决策书/四分析师/多空辩论/风控辩论），**弹窗 tab 与详情页长文共用，改一处两处同步**。
- `market.ts`：市场徽标（A股蓝/港股橙/美股绿）。
- 关键组件：
  - `stock-insight-modal.tsx`：个股洞察大弹窗（行情、K线、AI 建议、提醒面板、加仓计算器），是多数页面点击股票后的统一入口。
  - `InteractiveKline.tsx`：K 线图（依赖 CDN 的 `window.LightweightCharts`）；`KlineModal.tsx` 是其 Dialog 外壳。
  - `deep-analysis-modal.tsx`：深度分析三态弹窗（确认触发→进度轮询→结果展示）。
  - `suggestion-badge.tsx` / `ai-suggestion-badge.tsx` / `technical-badge.tsx` / `suggestion-action.ts`：AI 建议徽标与 action 归一化（buy/add/reduce/sell/hold/watch 五档 + 红涨绿跌配色）。
  - `kline-summary-dialog.tsx` / `kline-indicators.tsx`：K 线技术摘要。
  - `price-alert-form-dialog.tsx` / `stock-price-alert-panel.tsx`：提醒规则表单与面板。
  - `logs-modal.tsx`：全局日志查看器（配合 `src/lib/logger-map.ts` 把 Python logger 名翻译成中文）。
  - `onboarding.tsx`：新手引导；`AmbientBackground.tsx`：全局氛围背景。
- 注意：biz-ui 部分组件直接 import `@/lib/utils`、`@/lib/kline-scorer`（主应用的别名），**packages 并非完全自包含**，拆出去单独发布会断。

## 主应用（src/）

### 入口与路由

- `main.tsx`：`BrowserRouter` + `ToastProvider` 包裹 `App`。
- `App.tsx`（310 行）：单文件完成路由表、桌面/移动双套导航（顶部 sticky nav + 移动端底部 tab）、认证守卫、版本检查。
  - 路由：`/` Dashboard、`/portfolio` Stocks、`/opportunities`、`/paper-trading`、`/alerts`、`/agents`、`/history`、`/datasources`、`/settings`、`/analysis/:symbol/:date` 详情页、`/login`。
  - `RequireAuth`：只查 `localStorage` 有无 token（未做后端校验），无则跳 `/login`。
  - 登录后调用 `/settings/update-check` 做升级提醒，"稍后提醒"按版本号记入 `localStorage(panwatch_upgrade_dismissed_version)`。

### 主要组件（src/components/）

- `ChatWidget.tsx`：全局右下角 AI 聊天浮窗；**通过 `window` CustomEvent 接收个股上下文**（stock-insight-modal 里点"问问 AI"时派发）。
- `DiscoveryPanel.tsx`：首页"发现"面板（热门板块/个股），带内存缓存与 localStorage 记住的 tab/市场选择。
- `ShareCardDialog.tsx` + `ShareCardModal.tsx` + 4 张业务卡（Benchmark/Diagnostics/Digest/SignalScore）：用 `html-to-image` 的 `toPng` 导分享图；**卡片颜色必须显式内联十六进制**，不能依赖主题 CSS 变量（保证暗色主题下导出一致）。
- `SelfCheckModal.tsx`：系统自检（先拉清单秒渲染，再按 key 小并发逐项探测）。
- `AccountMenu.tsx`：头像下拉（折叠导航/主题切换/退出）。
- `FactorWeightsPanel.tsx`：因子权重编辑（权重/锁定/自动标定，展示最近 IC/IR）。

### hooks 与 lib

- `hooks/use-theme.ts`：亮/暗/跟随系统三态，`localStorage(panwatch-theme)` + 切换 `<html>` 的 class。
- `hooks/use-avatar.ts`：头像存后端（`/settings/avatar`），前端内存缓存 + CustomEvent 跨组件同步；上传图压缩成 128px JPEG data URL。
- `lib/utils.ts`：`cn` re-export、`useLocalStorage`（localStorage 持久化 state，广泛使用）、时间格式化三件套。
- `lib/kline-scorer.ts`：纯前端技术面打分器——把 K 线摘要（均线/MACD/RSI/KDJ/BOLL/量能/支撑压力）映射为分数和五档操作建议，**不经过后端/AI**，持仓与否走两套阈值。
- `lib/logger-map.ts`：Python logger 名→中文显示名映射（最长前缀匹配），新增后端模块时应同步补充。

## 核心机制 / 状态存储

- **状态管理**：无全局 store，全部 `useState` + props + 少量 CustomEvent（聊天上下文、头像变更）+ `useLocalStorage`（UI 偏好，键名多为 `panwatch_*`）。
- **认证**：JWT 存 localStorage（`token`、`token_expires`），`fetchAPI` 自动带 Bearer；401 全局登出。
- **主题**：CSS 变量 + `.dark` class；红涨绿跌的 A 股配色散落在各处显式 class（`text-rose-500` 涨 / `text-emerald-500` 跌）。
- **服务端状态**：全部即时拉取，无客户端缓存层；慢接口靠 `timeoutMs` 放宽（AI 类 40–120s）和 Dashboard 的快慢车道策略消化。

## 对外接口

- 前端消费的后端路由全部以 `/api` 为前缀（dev 由 Vite 代理，生产由 FastAPI 静态托管同源）；涉及约 60+ 端点，集中在 `/stocks`、`/recommendations/*`、`/dashboard/*`、`/paper-trading/*`、`/agents/*`、`/portfolio/*`、`/price-alerts/*`、`/datasources`、`/providers/*`、`/channels`、`/settings/*`、`/chat/*`、`/health/selfcheck`、`/auth/*`。
- 后端响应信封约定：`{code: 0, data, message}`，非 0 即异常。
- 新增页面的步骤：在 `pages/` 建组件 → `App.tsx` 注册 Route + navItems → 在 `packages/api` 加领域方法。

## 二次开发扩展点与注意事项

1. **Stocks.tsx（3134 行）是最大的坏味道**：类型与 `packages/api` 中的定义大量重复（Stock/Position/PortfolioSummary 等），改动持仓数据结构时需多处同步，建议优先抽类型。
2. **新增 API 必须走 `packages/api`**（README 硬性约定），否则破坏统一鉴权/超时/错误处理；唯一的原生 fetch 先例是 PDF blob 下载。
3. **K 线图依赖外网 CDN**：`window.LightweightCharts` 在 InteractiveKline 内部轮询等待加载；内网部署或构建产物审查时需把 lightweight-charts 改为 npm 依赖。
4. **改深度分析展示结构只改 `analysis-sections.ts`**，弹窗与详情页共用；改 section id 需同步 `AnalysisDetail.tsx` 的 `SECTION_ICON`。
5. **分享卡禁用主题变量**：导出 PNG 用显式色值，新增分享卡时复用 `ShareCardDialog` 外壳即可。
6. **配色方向与美股相反**：红=涨/看多（rose），绿=跌/看空（emerald），改样式时极易写反；`tailwind.config.js` 的 `stock.up/down` 与此一致。
7. **cron 解析是脆弱的**：Agents 页 `parseCronToConfig` 只识别少数模式（`*/N` 间隔、固定时间、工作日），复杂 cron 会落到"自定义"分支只读展示。
8. **新增通知渠道类型**改 `Settings.tsx` 的 `CHANNEL_TYPE_FIELDS`；**新增数据源类型/分类**改 `DataSources.tsx` 的 `DATASOURCE_TYPES` / `DATASOURCE_CATEGORIES`（含"其他"兜底）；**新增后端日志模块**补 `logger-map.ts`。
9. **biz-ui 不完全独立**：其组件引用主应用 `@/lib/*`（utils、kline-scorer、logger-map），移动组件到 biz-ui 时注意依赖方向不要成环。
10. **timeoutMs 默认值 20s 对 AI 接口不够**，新增慢接口记得显式放宽（参考 addPositionEval 60s、triggerAgent 120s）。
11. 未确认项：Service Worker 缓存策略是否会影响发版后静态资源更新（`CACHE_NAME='panwatch-v2'`，靠改版本号清缓存），本次未完整通读 `sw.js` 全部逻辑。
