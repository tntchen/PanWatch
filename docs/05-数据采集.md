# 05 · 数据采集层与数据源路由

> 面向二次开发者的结论性文档。范围：`src/collectors/`、`src/core/marketdata_client.py`、`src/core/data_collector.py`，以及 `server.py` 数据源注册段、`src/web/api/datasources.py`。

## 一句话总结

数据采集层是**薄 shim 层**：真正的抓取/主备调度/缓存/健康度全部收口在本地包 `packages/marketdata`（包名 `marketdata`）；host 侧只负责三件事——把 `DataSource` 表翻译成包的 `ConfigProvider`、把包返回的类型转成旧消费方同形结构、以及提供"数据源测试/日志"的管理 API。

## 架构总览（数据流）

```
FastAPI / Agents / 调度器
   │
   ├─ src/collectors/*.py        薄 shim：NewsItem/KlineData 等 host 结构 + 惰性 import
   │        │
   ├─ src/core/marketdata_client.py   接线层：DbConfigProvider + get_market_data() 单例
   │        │
   └─ packages/marketdata        包内：MarketData 门面 → Engine(主备链+唯一缓存层+Metrics)
              ↑ 配置来源：DbConfigProvider 现查 SQLite `data_sources` 表（每次取数现查）
```

关键设计事实：

- **`marketdata` 包是事实上的唯一数据源实现**。所有 collector 的"取数"最终都落到 `MarketData.{quotes,klines,news,events,capital_flow,flash_news,fundamentals,dragon_tiger,margin,shareholders,dividend,northbound}` 或不经 Engine 的 `hot_stocks/hot_boards/board_stocks`（东财 discovery 榜单）。
- **Engine = 主备链**：按 `DataSource.priority` 排序，首个非空结果即返回（`min_count` 不足会换源/取最长）；缓存只在 Engine 一层，vendor 内不各自缓存。
- **配置现查 DB**：`DbConfigProvider.sources_for()` 每次都新开会话查 `data_sources` 表，因此 UI 改数据源 enabled/priority/config **立即生效**，无需重启、无失效钩子（`get_market_data()` 单例无状态）。
- **vendor 合法性权威**：`marketdata.PACKAGE_VENDORS_BY_TYPE`（由包内 `registry.py` 的 vendor 类映射派生）∪ `server.DATA_SOURCE_SEEDS` 中的 provider 集合 = 合法集合，之外的 DB 行是"孤儿"。

## 数据源注册与健康检查

- **注册点（server.py）**：
  - `DATA_SOURCE_SEEDS`（约 354 行起）：全部预置数据源种子，13 种 type（news/kline/capital_flow/quote/events/chart/flash_news/fundamentals/dragon_tiger/margin/shareholders/dividend/northbound），每行含 name/type/provider/config/enabled/priority/supports_batch/test_symbols。新增预置源**改这里**。
  - `seed_data_sources(db=None)`：按 (name, provider) 只增不删 upsert；已存在行只补 `supports_batch` 和空的 `test_symbols`，**不覆盖用户改过的 config/priority/enabled**。
  - `reconcile_data_sources(db)`：启动时（约 1456 行）+ `POST /api/datasources/reset-to-seed` 调用。补缺失种子 + 删孤儿（`(type,provider)` 不在 `PACKAGE_VENDORS_BY_TYPE ∪ seed 集合`），其余行原样保留。
- **健康检查**：无主动探活。健康度 = 包内 Engine 每次取数经 `MetricsSink` 记录的内存快照（成功率/p50 延迟/最近错误），`MarketData.health()` 暴露；`GET /api/datasources` 列表时按 provider 挂到每行 `health` 字段。主动验证靠 `POST /api/datasources/{id}/test`（走 `DataCollectorManager.test_source()`，单 vendor 隔离测试，返回 `test_passed/error/items/logs`）。
- **API 路由（`src/web/api/datasources.py`，挂在 `/api/datasources`）**：CRUD + `/types` + `/reset-to-seed` + `/{id}/test`。每行响应带 `is_orphan` 与 `engine_attached`（type 是否已接入包引擎；chart 类型不在集合内）。

## 对外接口速查

- 给业务代码用：`get_market_data()`（包门面）、`md_quote_rows/md_stock_data/md_news/md_news_by_keyword`（同形 dict/dataclass）、各 Collector 类（agents/调度器直接实例化，如 `KlineCollector(MarketCode.CN)`）。
- 同步/异步约定：包与 shim 的取数函数**全是同步**；async 调用方一律 `await asyncio.to_thread(...)`。时间坑：`md.news()` 的 `now` 必须传 **aware UTC**（包内 publish_time 是 aware），`md_news()` 已处理好。
- 代理：全局走 env（`trust_env=True`，由 `apply_proxy_env` 按 UI http_proxy 设置）；个别源（Yahoo/YFinance）在 `source.config.proxy` 单独配置，默认关闭。

## 二次开发扩展点与注意事项

**新增一个数据源（vendor）的正确路径**：
1. 在 `packages/marketdata/src/marketdata/vendors/` 实现 Vendor 类，并在 `registry.py` 的 `VENDOR_CLASSES_BY_TYPE` 注册（`PACKAGE_VENDORS_BY_TYPE` 自动派生）。
2. 在 `server.py DATA_SOURCE_SEEDS` 加种子行（否则对账不会补、但也不会被当孤儿删——因为在包内集合里即合法）。
3. **不要**改 `src/collectors/`：包接入后 collector shim 自动走通。`data_collector.py` 的 `_test_xxx_source` 是按类型派发、自动支持新 vendor，无需改。
4. 新增**数据类型**（而非新 vendor）才需要动：包 Engine/门面、`marketdata_client` 转换函数、`TYPE_LABELS`、`_ENGINE_ATTACHED_TYPES`、seed、`_test_source_impl` 分支。

**易踩的坑**：

- **`EastMoneyEventsCollector` 构造参数是死参数**（timeout/verify_ssl/proxy 等仅为兼容签名保留，取数走包）；改它们不生效，真实配置走包内 vendor。
- **`DataCollectorManager.COLLECTOR_FACTORIES` 与主取数路径无关**（kline 工厂返回元组占位）；别以它为准判断系统支持哪些源，权威是 `PACKAGE_VENDORS_BY_TYPE ∪ DATA_SOURCE_SEEDS`。
- **K 线缓存语义**：正缓存命中要求"TTL 内且条数 ≥ need"；条数不足会固化负缓存（冷却窗口内返回陈旧/空），排查"K 线不更新"先看冷却标记（`clear_kline_cache()` 可清）。
- **`/api/datasources/{id}/test` 返回结构故意不用 `success/data` 顶层字段**（避免 ResponseWrapperMiddleware 拆解），前端按 `test_passed` 读。
- **news 类型的 vendor 集合**：雪球(xueqiu)/东财个股新闻(eastmoney_news)/东财公告(eastmoney) 都在包内 news vendor 集合里，但 `enabled` 默认有别（雪球默认关、需 cookie）；事件日历(events) 与公告共用东财 notices 端点但属不同 type。
- **截图不走包**：改 K 线截图逻辑只动 `screenshot_collector.py`；它依赖 Playwright 且对目标站前端结构（CSS selector/固定 clip 坐标）强耦合，前端改版即失效。
- **进程级单例**：`get_market_data()` 单例本身无状态、配置现查 DB，所以改 DB 不用 `reset_market_data()`；只有测试隔离才需要 reset。
- **市场级 vs symbol 级**：flash_news/dragon_tiger/northbound 是市场级（测试不传 symbols，dragon_tiger 需 `config.test_date`）；fundamentals/margin/shareholders/dividend 按 symbol（测试必须配 test_symbols，否则直接报"请配置测试股票代码"）。
