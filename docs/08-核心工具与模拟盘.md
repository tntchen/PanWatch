# 08 · 核心工具与模拟盘（src/core/）

> 范围：模拟盘引擎、组合诊断、入场候选、信号解释、盘中事件门控、新闻排序、K线上下文、上下文构建、JSON 存储、链接生成、系统自检、日志上下文、Agent 目录。
> 面向二次开发者，重结论轻罗列。

## 模块职责一句话

这一组文件是 PanWatch 的「工具层 + 模拟交易闭环」：**EntryCandidate（入场候选）按日快照打分出买卖计划 → StrategySignalRun（策略信号）驱动 PaperTradingEngine 自动开平仓 → portfolio_diagnostics 只读诊断持仓集中度**；其余文件是支撑 Agent 链路的纯函数工具（上下文构建、新闻排序、事件门控、链接、自检、日志、Agent 种子配置）。

## 核心机制与数据流

### 模拟盘（paper_trading_engine）

- **入口**：`paper_trading_scheduler.py`（定时）与 API `src/web/api/paper_trading.py` 都调 `ENGINE.scan_once()`；同步逻辑 `_scan_sync` 在 `asyncio.to_thread` 里跑，避免阻塞事件循环。
- **建仓**：查 `StrategySignalRun` 中 `status=active` 且 `action in (buy, add)` 的信号（rank_score 降序取前 50，同股票去重、已有持仓跳过）→ 批量取报价（`md_quote_rows`，走 QuoteOrchestrator 多 provider 故障转移）→ 按 `_position_weight(rank_score)` 分档（85+ 投 25%、65-75 投 12% 等）× 市场预算算股数（100 股整手，含买入费不超子池现金）→ 用当前市价入场。
- **分市场子池**：账户级 `market_allocations`（默认 CN 50% / HK 30% / US 20%），某市场可用现金 = `总资金×比例 + 该市场已实现盈亏 − 该市场持仓成本`（纯函数 `compute_market_cash`）。比例为 0 的市场不建仓。`allocations_from_excluded` 是迁移辅助（被 `src/web/migrations.py` 使用）。
- **平仓**：依次检查 固定止损 → 固定止盈 → 移动止损（浮盈 ≥5% 后从最高价回撤 10%）→ 信号反转（同股出现 active 的 sell/reduce 信号）→ 时间止损（默认 20 自然日，信号带 `holding_days` 时优先）。盈亏含双边成本（与回测共用 `src/core/backtest/cost_model.py` 的 `CostModel`），保证资金守恒。
- **通知**：开平仓事件在 DB session 关闭前先序列化为 plain dict（`_serialize_*`，防 ORM detached），再异步调 `paper_trading_notifier.notify_entry/notify_exit`，通知失败不影响交易。
- **状态存储（全部在 SQLite）**：`PaperTradingAccount`（单行，初始 100 万）、`PaperTradingPosition`（open/closed）、`PaperTradingTrade`（已平仓记录）、`StrategySignalRun`（上游信号，由 strategy_engine / tradingagents 写入）。

### 入场候选（entry_candidates）

- **两路来源**：① watchlist 路——最新未过期的 `StockSuggestion`（Agent 建议）；② market_scan 路——东财热榜（成交额榜 + 涨幅榜，经 `EastMoneyDiscoveryCollector`），不足时三级回退：近 7 天历史候选 → 近 14 天 `MarketScanSnapshot` → 硬编码种子池 `MARKET_SCAN_SEED_SYMBOLS`。
- **打分**：`ACTION_BASE_SCORE`（buy=78 / add=72 … sell=20）为基底，叠加 K 线形态（均线排列 ±8、MACD ±5、量比、RSI/KDJ）、上下文质量分、新鲜度加减分，clamp 到 [0,100]。
- **交易计划**：`_build_plan` 用现价 ±1% 作入场区间、支撑位下方作止损、阻力位作止盈，兜底 -5%/+6%；`_plan_quality` 给计划完整度打分（四项各 30/30/30/10）。
- **激活门槛**：`action in (buy,add)` 且 `plan_quality>=90` 且 score ≥ 阈值（market_scan/mixed 62，watchlist 55）→ `status=active`，否则 `inactive`。
- **落库方式**：每次 `refresh_entry_candidates` 按 `snapshot_date` **先删后插**（当日全量重写），同日重复刷新不会产生重复行。反馈（`EntryCandidateFeedback`）与后验（`EntryCandidateOutcome`，T+1/3/5/10 收益与止盈止损命中）是独立表，评估分批 commit（每 50 条）缩短 SQLite 写锁窗口。
- **调用方**：`strategy_engine.py` 定时调用；API 在 `src/web/api/recommendations.py`（list 时可惰性触发 refresh）。

### 上下文构建（context_builder）

被 premarket_outlook / intraday_monitor / daily_report 三个 Agent 及 `api/agents.py` 复用。每个 symbol 产出 payload：`technical_current / kline_history / relative_strength(vs 沪深300·恒指·标普500 的 5/20 日超额收益) / ta_verdict(14天内 TradingAgents 结论) / news{realtime,extended,history,history_topic} / events(重要公告附东财全文,top3,≤1000字) / constraints(持仓/资金/risk_budget_hint) / memory(近30天质量分趋势) / data_quality(0-100 覆盖度评分)`。K 线与指数上下文在单次构建内缓存。快照经 `context_store` 落库供「记忆」复用。

## 对外接口

- **API 路由**：`src/web/api/paper_trading.py`（模拟盘 CRUD + 手动平仓 + `diagnose_paper_portfolio`）、`src/web/api/recommendations.py`（候选列表/刷新/反馈/后验/统计，`enrich_signal` 在此注入）、`src/web/api/health.py`（`list_selfcheck_items` / `run_selfcheck`）、`src/web/api/accounts.py`（`diagnose_positions` 复用为真实账户诊断）。
- **注册点**：`server.py` 用 `AGENT_SEED_SPECS` 播种 Agent 配置（`seed_agents`）、`install_log_record_factory()` 启动时安装日志工厂、`log_context(...)` 包裹 Agent 执行；`paper_trading_scheduler.py` 注册模拟盘定时扫描。
- **配置项**：AppSettings keys `stock_link_platform`、`panwatch_base_url`；账户字段 `market_allocations`；intraday_monitor Agent config 的 `price_alert_threshold` / `volume_alert_ratio` 等（见 agent_catalog 种子）；`DATA_DIR` 环境变量决定事件门控状态文件位置（`$DATA_DIR/state/intraday_monitor_state.json`）。

## 二次开发扩展点与注意事项

1. **新增平仓/建仓规则**：改 `paper_trading_engine.py` 的 `_check_exits` / `_check_entries`；常量（TRAILING_STOP_PCT、止损止盈兜底 -8%/+15%、仓位分档 `_position_weight`）都是模块级硬编码，想做成可配置需自己引入设置项。
2. **SQLite 写锁是本项目反复踩的坑**：`_check_exits` 里信号查询特意包 `db.no_autoflush`、候选后验分批 commit、selfcheck 的 hint 也专门处理 "database is locked"——新增 DB 写路径时遵守「扫描中途不 flush、统一末尾 commit」的既有模式。
3. **ORM detached**：跨 session 边界传递持仓/交易对象前必须用 `_serialize_*` 转 dict；新增事件通知路径时照做。
4. **EntryCandidate 按日整表重写**：不要假设行 id 跨刷新稳定；关联要用 `(snapshot_date, stock_symbol, stock_market)`。打分阈值（55/62）、种子池、东财榜单均为硬编码，改来源需动 `_load_market_scan_inputs` 的三级回退链。
5. **新增 Agent**：在 `AGENT_SEED_SPECS` 加种子 + server.py `AGENT_REGISTRY` 注册；kind 由名字推断（capability 白名单在 `agent_catalog.py`），默认一律 workflow——新 capability Agent 必须加进 `CAPABILITY_AGENT_NAMES`。
6. **stock_link 目前只支持雪球**：`PLATFORMS` 字典是扩展点，但 `stock_url` 的非雪球分支直接兜底雪球，加平台要同时改映射与 fallback。
7. **fail-soft 约定**：kline_context、context_builder 的相对强度、analysis_link、news_ranker 等全部「取不到就降级不抛错」；扩展时保持该约定，否则 Agent 主链路会被数据源故障击穿。
8. **时区**：`news_ranker.parse_news_time` 统一转本地 naive datetime 与 `datetime.now()` 比较；`paper_trading_engine` 用 UTC aware。两处风格不同，混用时间字段时注意 tzinfo。
9. **未读到的部分**：`paper_trading_notifier.py`、`cost_model.py`、`context_store.py`、`marketdata_client.py`、`scheduler_registry.py` 在范围外，仅确认了调用关系，未深读其内部实现。
