# PanWatch 架构总览：后端入口与基础设施

> 面向二次开发者。本文覆盖 `server.py`、`src/web/`（app / database / models / migrations / response / log_handler）、`src/config.py`、`src/models/`、`.env.example`、`config/watchlist.yaml`。其他模块（agents、collectors、api 路由细节）见后续文档。

## 一、模块职责一句话总结

- **server.py**：统一服务入口，负责启动生命周期（init_db / 代理 / SSL / Playwright / 种子数据）、Agent 注册与调度、AI 模型/通知渠道解析、手动触发 Agent，并挂载 FastAPI app 与 SPA 静态文件服务。
- **src/web/app.py**：纯路由装配层——创建 FastAPI 实例、注册中间件、按"公开/需登录"两类挂载 27 个 API 路由。
- **src/web/database.py**：SQLAlchemy 引擎与会话工厂（SQLite WAL），含 init_db 及历史数据迁移（旧 provider/旧设置/旧持仓/旧软删除字段）。
- **src/web/models.py**：全部 ORM 模型定义（30+ 张表），是数据库 schema 的唯一事实来源之一（与 migrations 互补）。
- **src/web/migrations.py**：版本化迁移框架（v101–v118），带 `schema_migrations` 账本与 checksum 校验。
- **src/web/response.py**：统一响应包装中间件，把 `/api/` 下所有 JSON 响应包成 `{code, success, data, message}`。
- **src/web/log_handler.py**：异步缓冲写库的 logging handler，把日志持久化到 `log_entries` 表供 UI 日志板查询，并做保留期清理。
- **src/config.py**：环境变量配置（pydantic-settings `Settings`）+ YAML watchlist 加载 + `AppConfig` 组合对象。
- **src/models/market.py**：市场领域模型（MarketCode 枚举、三市场交易时段/时区/代码正则、StockData/IndexData 数据类）。

## 二、整体架构

```
uvicorn (port 8000)
 └─ server.py: lifespan
     ├─ init_db()          → database.py + migrations.py
     ├─ setup_logging/proxy/ssl/playwright
     ├─ seed_agents / reconcile_data_sources / seed_strategies / seed_sample_stocks
     ├─ build_scheduler().start()          ← AgentScheduler (APScheduler)
     ├─ PriceAlertScheduler (60s 轮询)
     ├─ PaperTradingScheduler (60s 轮询)
     └─ ContextMaintenanceScheduler (6h 评估)
 └─ src/web/app.py: FastAPI
     ├─ ResponseWrapperMiddleware → 统一 {code, success, data, message}
     ├─ /api/auth, /api/market    → 无需登录
     └─ 其余 ~25 个路由           → Depends(get_current_user)
 └─ static/ (若存在) → SPA 兜底路由 serve_spa
```

技术栈：FastAPI + SQLAlchemy 2.x + SQLite（WAL, NullPool）+ APScheduler（封装在 src/core/scheduler.py）+ pydantic-settings。单进程单体应用，所有状态（除 watchlist.yaml / .env / static）都在 SQLite。

## 三、启动流程（lifespan 精确顺序）

1. `init_db()`：`Base.metadata.create_all` → `_migrate()`（非版本化增量加列）→ `_migrate_old_providers`（ai_providers → ai_services+ai_models）→ `_migrate_settings_to_models`（app_settings 里的 AI/Telegram 配置迁移为 AIService/AIModel/NotifyChannel 并删除旧 key）→ `_migrate_positions_to_accounts`（stocks 表持仓迁入 accounts+positions）→ `_migrate_remove_stock_enabled` → 若有待执行版本化迁移先 `_backup_db_before_migration()`（生成 `panwatch.db.bak.<时间戳>`）→ `run_versioned_migrations`。
2. `setup_logging()`：root DEBUG；控制台 handler 按 `LOG_LEVEL` 过滤并用 `_ConsoleNoiseFilter` 丢弃 httpx/httpcore/urllib3/apscheduler/uvicorn.access 的 <WARNING 噪音；`DBLogHandler` 全量 DEBUG 入库；强制 uvicorn 三个 logger propagate 到 root。
3. `setup_proxy()`：优先级 = 已有环境变量 > `app_settings.http_proxy` > `.env` 的 `http_proxy`，最终写入进程级 `HTTP_PROXY/HTTPS_PROXY/NO_PROXY`，所有 httpx（trust_env=True）生效。
4. `setup_ssl()`：若配置企业 CA，则把 certifi bundle + 企业证书拼成 `data/ca-bundle.pem`，设 `SSL_CERT_FILE`/`REQUESTS_CA_BUNDLE`。
5. `setup_playwright()`：Docker（`DOCKER=1`）下浏览器装到 `$DATA_DIR/playwright`；本地用系统安装；可用 `PLAYWRIGHT_SKIP_BROWSER_INSTALL=1` 跳过。
6. `init_auth_from_env(db)`：从 `AUTH_USERNAME`/`AUTH_PASSWORD`/`JWT_SECRET` 等环境变量初始化认证（实现见 src/web/api/auth.py，未深读）。
7. `seed_agents()` → `reconcile_data_sources()` → `seed_strategies()` → `seed_sample_stocks()`（仅当 stocks 表为空时插 5 只示例股）→ TradingAgents 建议回填（幂等）→ 后台线程刷新股票列表缓存（src/web/stock_list.py）。
8. 启动 4 个调度器；yield 后逆序 shutdown。

## 四、Agent 注册与调度机制

**双层结构：代码层注册表 + 数据库层配置。**

- **代码层** `AGENT_REGISTRY: dict[str, type]`（server.py:1107）：`name → Agent 类`，当前 6 个：daily_report、premarket_outlook、news_digest、chart_analyst、intraday_monitor、tradingagents。
- **种子层** `AGENT_SEED_SPECS`（src/core/agent_catalog.py）：`AgentSeedSpec` 冻结数据类，声明每个 Agent 的 display_name / schedule / execution_mode / kind / visible / config 等。`seed_agents()` 每次启动 upsert 到 `agent_configs` 表：**始终同步** execution_mode / display_name / kind / visible / lifecycle_status，但**不覆盖**用户已有的 enabled / schedule / config（仅空时补齐）；`kind != "workflow"` 的 Agent 被强制 `enabled=False, schedule=""`（capability 型不参与调度）。
- **调度**：`build_scheduler()` 只注册 `enabled=True AND kind='workflow'` 且有 schedule 的 Agent；`AgentConfig.config` 作为构造 kwargs 传入（TypeError 时回退无参构造）。时区取 `Settings.app_timezone`（TZ 环境变量，默认 Asia/Shanghai）。
- **上下文构建** `build_context(agent_name)`：watchlist（来自 stock_agents 关联表，**不是** watchlist.yaml）→ 多账户持仓（accounts+positions）→ 代理 → 三级 AI 模型解析（stock_agent 覆盖 → agent 默认 → `AIModel.is_default` → 第一个）→ 三级通知渠道解析（同上三级，空则取 `is_default` 渠道）→ 组装 `AgentContext`（含 AIClient / NotifierManager / AppConfig / PortfolioInfo / model_label）。
- **执行模式**：`execution_mode` ∈ `batch`（全量股票一次 `agent.run(context)`）/ `single`（逐只 `agent.run_single`）。手动触发走 `trigger_agent` / `trigger_agent_for_stock`，后者对 intraday_monitor / tradingagents 有特殊参数注入（bypass_throttle、`_trace_id`、`_force_refresh` 等 setattr 到 context）。
- **运行记录**：每次执行通过 `record_agent_run` 写入 `agent_runs` 表（status / trace_id / trigger_source / notify 状态 / duration_ms / model_label）。

## 五、数据模型与数据库

**数据库**：`data/panwatch.db`（SQLite，WAL + busy_timeout=30s + synchronous=NORMAL + foreign_keys=ON，NullPool 每次新建连接）。FastAPI 依赖注入用 `get_db()`；后台/调度代码直接 `SessionLocal()`。

**核心表分组**（models.py，约 30 张）：

| 分组 | 表 | 说明 |
|---|---|---|
| AI 配置 | ai_services, ai_models | 服务商（base_url+api_key）一对多模型，`is_default` 兜底 |
| 通知 | notify_channels | type+config(JSON)，如 telegram 的 bot_token/chat_id |
| 股票/持仓 | stocks, accounts, positions | stocks 仅存 symbol/name/market/sort_order（持仓字段已废弃但列还在）；positions 按 (account_id, stock_id) 唯一 |
| Agent 绑定 | stock_agents, agent_configs, agent_runs | stock_agents 是股票×Agent 多对多，可挂 ai_model_id/notify_channel_ids 覆盖 |
| 日志/设置 | log_entries, app_settings | app_settings 是通用 KV 表（http_proxy、notify_quiet_hours 等） |
| 数据源 | data_sources | type/provider/priority/supports_batch/test_symbols，运行时按 type+priority 选源 |
| 分析/记忆 | analysis_history, stock_context_snapshots, news_topic_snapshots, agent_context_runs, news_cache | 跨天上下文记忆与新闻去重（source+external_id 唯一） |
| 建议/策略层 | stock_suggestions, entry_candidates(+feedback/outcomes), market_scan_snapshots, strategy_catalog, strategy_signal_runs, strategy_outcomes, strategy_weights(+history), factor_weights(+history), strategy_factor_snapshots, market_regime_snapshots, portfolio_risk_snapshots | "建议池 → 候选榜 → 策略信号 → 后验评估 → 动态调权"完整链路 |
| 价格提醒 | price_alert_rules, price_alert_hits | 条件组 JSON、冷却/每日次数/过期；hits 按 trigger_bucket 分钟级去重 |
| 模拟盘 | paper_trading_account(单例), paper_trading_positions, paper_trading_trades | market_allocations 各市场投资比例 |
| 对话 | chat_conversations, chat_messages | AI 对话 |
| 其他 | notify_throttle（通知节流）、suggestion_feedback | |

**迁移三层体系**（重要，踩坑高发）：
1. `Base.metadata.create_all`：只建新表，不改旧表。
2. `database.py::_migrate()`：非版本化的 `ALTER TABLE ADD COLUMN` 列表 + 悬空 FK 清理（`_drop_dangling_ai_provider_fk`）+ 数据型迁移（旧 providers/旧 settings/旧持仓/stocks.enabled）。
3. `migrations.py` 版本化迁移 v101–v118：记录到 `schema_migrations(version, name, checksum, success, error)`；checksum 取自 runner 源码哈希，**改了已发布迁移的代码会触发重跑**（ON CONFLICT 重置 success=0 后重新执行）。

**状态存储位置速查**：几乎全部业务状态在 SQLite；`config/watchlist.yaml` 仅供 `load_config()` 读取（当前主链路实际从 DB 的 stock_agents 加载 watchlist，YAML 更接近遗留/示例配置）；`.env` 是环境变量兜底；`data/` 下还有 ca-bundle.pem、playwright 浏览器、股票列表缓存（src/web/stock_list.py）；内存状态 = 4 个调度器实例 + DBLogHandler 缓冲区 + uvicorn 进程。

## 六、对外接口

- **API 路由**（app.py）：公开 = `/api/auth/*`、`/api/market/*`、`/api/health`、`/api/version`；其余 25 个路由（stocks/quotes/klines/insights/agents/providers/channels/datasources/settings/logs/history/context/news/suggestions/templates/feedback/discovery/price-alerts/recommendations/dashboard/factors/health/paper-trading/chat/accounts）均需 `get_current_user`。注意 `app.py:176` 的 `/api/health` 与 health 路由同前缀并存。`redirect_slashes=False` 是为避免重定向丢 Authorization header。
- **响应协议**：`{code, success, data, message}`。2xx 默认包成功；业务层返回含 `success` 字段时按之规范化；非 2xx 取 `detail` 组装失败响应。中间件是纯 ASGI 实现（规避 BaseHTTPMiddleware 的 streaming hang）。
- **注册点**：新 Agent → `AGENT_REGISTRY` + `AGENT_SEED_SPECS`；新数据源 → `DATA_SOURCE_SEEDS`（server.py:354，20 条预置，覆盖 news/kline/quote/capital_flow/events/flash_news/fundamentals/dragon_tiger/margin/shareholders/dividend/northbound/chart 类型）；新 API → src/web/api/ 下建路由并在 app.py 挂载。
- **配置项优先级**（通用规律）：UI 设置（app_settings 表 / AIService / NotifyChannel / AgentConfig）> `.env`（Settings）> 代码默认值。代理是例外：显式环境变量 HTTP_PROXY 最高优先。
- **环境变量**（.env.example + Settings）：AI_BASE_URL/AI_API_KEY/AI_MODEL、NOTIFY_TELEGRAM_*、HTTP_PROXY、CA_CERT_FILE、DAILY_REPORT_CRON、TZ(=app_timezone)、LOG_LEVEL、DATA_DIR、DOCKER、AUTH_USERNAME/AUTH_PASSWORD/JWT_SECRET（见 auth.py）、PLAYWRIGHT_*、DEV_RELOAD。

## 七、二次开发扩展点与注意事项

**扩展点**
1. **加 Agent**：实现 `src/agents/` 下的类（继承 base，提供 `run(context)`，可选 `run_single`）→ 注册 `AGENT_REGISTRY` → 加 `AgentSeedSpec`（决定 kind/schedule/config 默认值）→ prompts/ 下加提示词文件。注意 seed 不会覆盖用户已改的 enabled/schedule/config，改默认值只对新装生效。
2. **加数据源**：在 `DATA_SOURCE_SEEDS` 加一条（name+provider 为 upsert 键）；`reconcile_data_sources` 会删除 `(type, provider)` 不在 seed ∪ `marketdata.PACKAGE_VENDORS_BY_TYPE` 的"孤儿"行——**新增 provider 若不在 marketdata 包的合法集合里，必须进 seed，否则启动时被删**。
3. **加表/改 schema**：新表只需在 models.py 加模型（create_all 自动建）；改已有表必须走 migrations.py 新版本号（不要改已发布迁移函数，checksum 机制会导致重跑甚至线上数据被二次改写）；`database.py::_migrate()` 的裸 ADD COLUMN 是历史遗留风格，新代码应优先用版本化迁移。
4. **加 API**：src/web/api/ 新建模块，app.py 挂载；默认需登录（dependencies=protected），公开接口需显式不加。

**注意事项 / 坑**
- `init_db()` 中的数据型迁移（`_migrate_*`）都是幂等但无版本记录的，依赖 `_has_table/_has_column` 探测；自定义改过 schema 的数据库行为可能出乎意料。
- `Stocks.cost_price/quantity/invested_amount` 列仍在但已废弃，持仓真相在 `positions` 表；读持仓请用 `load_portfolio_for_agent` 的口径。
- `resolve_ai_model` / `resolve_notify_channels` 每次调用都新建 Session 并 `expunge` 返回对象，返回的是游离对象——别在闭包/异步任务里再次访问其 relationship。
- `ResponseWrapperMiddleware` 缓冲整个响应体，大文件/流式响应若走 `/api/` 前缀会被整体缓存（目前文件下载路由需确认是否避开 /api/ 前缀）。
- DBLogHandler 缓冲上限 2000 条、超量丢最早的（计入 dropped），ERROR 级立即 flush；保留策略：基础设施日志 3 万条、总量 12 万条。`get_log_handler_stats()` 可查健康度。
- 生产模式 `python server.py` 不开 reload（避免 reloader 子进程和 data/ 写入误触发）；热重载用 `make dev-api` 或 `DEV_RELOAD=1`。
- 静态 SPA 兜底路由 `@app.get("/{path:path}")` 仅在 `static/` 存在时注册，开发环境（前端 vite :5183）不存在，不影响 API。
- seed_agents 会把 capability 类 Agent 强制禁用并清空 schedule；若想让某 Agent 重新参与调度，其 kind 必须是 workflow。
- SQLite `PRAGMA foreign_keys=ON` 已开启，删除 stocks/accounts 会级联删 positions/stock_agents/price_alert_rules——UI 删除操作有连带影响。

**未深读/不确定**：`src/web/api/*` 各路由实现、`src/core/*`（scheduler、notifier、ai_client、agent_runs、agent_catalog 的完整 seed 列表）、`marketdata` 包（外部依赖，`PACKAGE_VENDORS_BY_TYPE` 来源）、`src/web/stock_list.py` 缓存细节、auth.py 的 JWT 细节。这些在对应专题文档中展开。
