# 跟踪简报数据能力核查（Kimi 数据点 × PanWatch marketdata）

> 日期：2026-07-22 ｜ 核查方式：explore 子代理只读核查（对照 finance_project 跟踪模板与 packages/marketdata 真实代码）
> 用途：回答"PanWatch 跟踪简报能否达到 Kimi 同等效果"的输入侧问题，结论已回填 doc/12 v1.6

## 1. 数据点覆盖表

| 数据点 | 判定 | 依据 / 说明 |
|---|---|---|
| 现价/涨跌幅/成交量/成交额/量比 | ✅ | Quote 全字段（types.py:28-48）；腾讯/东财双源均填 |
| 换手率 turnover_rate | ✅ | Quote 独立字段（types.py:43），腾讯/东财双源；与成交额 turnover 已区分 |
| 日 K 线序列 | ✅ | klines() 返回 list[Bar]（client.py:131-137）；缺口/5 日线/平台/连续 N 日均可算。注意 Bar **无成交额**，"缩量"只能按成交量判断 |
| 大盘指数行情 | ✅ | index_quotes()（client.py:154-161），daily_report 已用；加科创50 是一行改动 |
| 科创50 日 K | 🟡 | index_klines() 依赖 INDEX_SECID 映射（client.py:36-42），**未含 000688**，未映射返回 [] |
| 存储板块行情 | 🟡 | 有通道：hot_boards() 行业榜 + board_stocks() 成分（discovery.py:95-184），但**无按板块代码定向取行情/K线的接口** |
| 主力资金净流入（逐股） | ✅ | capital_flow()（client.py:171-176），东财主源含主力/超大/大/中/小单 + 5 日主力；新浪备源 |
| 龙虎榜 | 🟡 | 市场级按日期快照（client.py:292-296），个股过滤=宿主侧按 symbol 筛（零成本）。字段：上榜原因/收盘/涨跌幅/净买/买入额/卖出额/换手率——**无席位明细**（买一~买五、机构/量化席位），"量化还是散户接力"只能推断 |
| 融资余额（逐股） | 🟡 | 通道在：margin() 逐股东财接口（market_flow.py:139-166），但 vendor 请求 30 行**只返回 rows[0]**，周度趋势被丢弃 |
| 个股新闻/公告（业绩预告类） | ✅ | news() 聚合雪球+东财新闻+东财公告（client.py:194-267），业绩预告 importance=3；events() 结构化事件覆盖业绩预告 |
| 行业新闻（存储合约价类） | ✅ | news_by_keyword() 东财全站搜索（client.py:269-274）；限制：每词 15 条、无 since 过滤 |
| Agent web 检索 | ❌ | AIClient 是纯 OpenAI 兼容 chat（ai_client.py:28-79）；chat_with_tools 仅 5 个本地工具，**无搜索工具，Agent 完全依赖本地数据** |

## 2. 缺口清单与补齐难度

| 缺口 | 难度 | 说明 | 处置（doc/12 v1.6） |
|---|---|---|---|
| 科创50 日 K 映射 | 小 | INDEX_SECID 加一行 `"000688": "1.000688"` | 纳入 P3e |
| 融资余额周度序列 | 小 | margin vendor 已抓 30 行，改为返回全部而非 rows[0] | 纳入 P3e |
| 存储板块定向行情 | 小~中 | 补东财板块 quote/K线端点（secid=90.BKxxxx），或调用侧先 hot_boards 定位代码 | 纳入 P3e |

| 龙虎榜席位明细 | 中 | 需新增东财席位接口（RPT_BILLBOARD_DAILYDETAILSBUY/SELL），包内加 vendor | 待用户决策（D10），可后置 |
| Agent web 检索 | 中 | 需接搜索 API 或给 chat_with_tools 挂搜索工具；否则用关键词新闻近似 | 待用户决策（D10），可后置 |

## 3. 结论

三大简报核心输入（实时行情含量比/换手、日 K 技术位、主力资金、个股新闻/公告、行业关键词新闻、大盘指数）**均现成可用**。影响复盘深度的缺口两处：龙虎榜席位级明细、融资余额周度趋势（后者补齐工作量小）。PanWatch Agent 无 web 检索，"卖方一致预期上修、长鑫挂牌动态"类外部信息只能用 news_by_keyword 关键词新闻近似替代。
