## 1. 分 Phase 验收方案

### P1 持仓交易化

**单元测试**（新增 `tests/test_position_trades.py` + `tests/test_position_cost_engine.py`）：
- 成本纯函数：加仓加权平均；减仓成本不变且 `realized_pnl = (卖价−成本)×数量−费`；减至 0 关仓保留行；**减仓数量 > 持仓必须 400**；关仓再买入走 trades 端点且成本重算正确；adjustment 流水字段完整；手续费 0 / 大额两边界。
- 精度两级断言：**Decimal 层精确相等；float 读回层容差 ≤0.001**（锚点：1600 股 + 两笔 → 3150 股 @112.572）。
- API 层：复用 `test_portfolio_result_cache.py:24-30` 内存 sqlite 模式；注入异常验证「流水+成本+资金」单事务回滚后三处都不变；有流水持仓 DELETE 返回 4xx。

**回归风险**：`update_position`（accounts.py:310）改为生成 adjustment 流水后，依赖"直接改数字"的旧行为会断；`delete_position` 级联语义变化。必跑 `test_portfolio_*`（result_cache / diagnostics / benchmark）+ `test_add_position_eval.py`；`get_portfolio_summary` 补「已实现盈亏」字段断言。

**人工验收**：UI 录入三笔历史流水 → 成本显示 112.572；减 500 股后流水/成本/资金三处一致；流水弹窗窄屏可用。

## 2. 测试数据与环境策略

- 东芯三笔历史流水用**真实数字做断言常量**（验收锚点），走合成持仓录入，不依赖生产库；v3.1 档案用脱敏精简版合成 payload 测 loader，完整档案只进人工验收。
- 一律内存 sqlite；**禁止用例碰 DATA_DIR 生产卷**；conftest `_ensure_db_schema`（:75-87）只建表不写数据。
- LLM mock 边界：**测到 prompt 组装为止**（断言摘要出现/不出现、模板变量填充），AI 响应 AsyncMock 固定返回；通知断言走 conftest 的 mock notifier。

## 5. 对研发评审测试策略的修正（doc/13 §2）

- "prompt 组装断言"必须升级为**无档案 prompt 逐字快照**。
- 研发策略漏了**迁移层**：checksum 机制下对已发布迁移的误改不会被现有测试发现，必须补旧库冒烟。
- "参照 test_price_alert_volume_ratio.py 模式"对 consecutive_close 不可行：该模式 mock 的 summary dict 不含真实 `{"error": ...}` 分支，照抄会漏边界，必须显式补空数据用例。
- "容差写进测试"细化为两级：Decimal 层精确相等、float 读回层 ≤0.001。
